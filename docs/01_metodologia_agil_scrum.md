# Metodologia agil aplicada: Scrum

## Metodologia seleccionada

Se selecciono Scrum porque permite organizar el desarrollo en incrementos cortos, priorizar funcionalidades de mayor valor y revisar el avance con entregables verificables.

## Roles

| Rol | Responsabilidad |
| --- | --- |
| Product Owner | Define prioridades, valida historias de usuario y acepta entregables. |
| Scrum Master | Facilita el proceso, elimina impedimentos y cuida el cumplimiento del sprint. |
| Developer 1 | Implementa backend, servicios, controladores y pruebas. |
| Developer 2 | Implementa vistas, layout administrativo y experiencia de usuario. |
| Developer 3 | Documenta, valida pruebas funcionales y prepara evidencias academicas. |

## Duracion del sprint

Sprint de 1 semana.

## Sprint Goal

Adecuar FarmaGo para demostrar una aplicacion funcional con modulos principales conectados, reglas criticas de inventario corregidas, manejo de excepciones y documentacion academica.

## Sprint Planning

Durante la planificacion se priorizaron:

- Cierre del registro publico.
- CRUD de productos, clientes y proveedores.
- Gestion de lotes.
- Registro de compras y ventas.
- Kardex y reportes.
- Documentacion de IA, excepciones, pruebas y refactorizacion.

## Daily Scrum

El equipo revisa diariamente:

- Que se hizo ayer.
- Que se hara hoy.
- Que impedimentos existen.

## Sprint Review

Se presenta una version funcional con login, dashboard, CRUDs, ventas, comprobante, Kardex, reportes y documentacion.

## Sprint Retrospective

Aspectos positivos:

- Se aprovecho el backend existente.
- Se conectaron modulos clave sin rehacer la arquitectura.
- La IA ayudo a detectar riesgos y acelerar documentacion.

Mejoras:

- Ampliar pruebas de interfaz.
- Agregar exportacion PDF/Excel en reportes.
- Mejorar permisos finos por modulo.
